Tarjan 割点学习笔记

发表于 2018-04-02 | 分类于 Algorithm

在一个无向图中，如果删掉点 后图的连通块数量增加，则称点 为图的割点。

Tarjan 算法可以在线性时间求无向图的所有割点。

定义

表示进入节点 时的时间（时间截）。

表示由节点 开始搜索所能到达的点中，在搜索树上是 的祖先且 最小的节点的 。

描述

类似于 Tarjan 求强连通分量，大致流程：

  1. 选定一个节点作为根节点，开始 DFS；
  2. 初始化当前点的 和 均为当前时间截；
  3. 遍历当前点 的所有邻接点（与 相连的点）；
  4. 如果某个邻接点 在 DFS 树中，更新 ；
  5. 如果某个邻接点 不在 DFS 树中，则对 进行 DFS，完成后更新 ；
  6. 对于根节点，如果它在 DFS 树上有两个或更多的子节点，则它为割点；
  7. 对于一个 DFS 树上的非根节点 ，如果存在子节点 ，满足 ，则 为割点。

解释

对于根节点，如果它在 DFS 树上有两个或更多的子节点，则它为割点；

显然，根是两棵子树上节点的唯一连通方式，删掉后破坏原图的连通性。
